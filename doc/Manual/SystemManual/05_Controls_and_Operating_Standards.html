Controls and Operating Standards

Back to Table of Contents
Core policy: Create -> Approve -> Post -> Reconcile -> Close.

Daily Standards

  1. Review approval and to-do queues from dashboard.
  2. Clear pending operational postings in Sales, Purchases, Receivables, and Payables.
  3. Review open allocations and unresolved exceptions.
  4. Validate critical balances in cash/bank and inventory.

Weekly Standards

  1. Review customer and vendor ageing outputs.
  2. Review approval turnaround and rejected transactions.
  3. Review role and permission changes.

Month-End Standards

  1. Complete document posting and approvals.
  2. Finalize bank reconciliation and voucher controls.
  3. Reconcile subledgers against General Ledger.
  4. Run financial reports and variance checks.
  5. Close month only after sign-off.

High-Risk Control Points

  • Unauthorized page access from incorrect role/token mapping.
  • Duplicate posting from repeated submit attempts.
  • Posting into periods that should be closed.
  • Over-allocation and unmatched receipt/payment balances.
