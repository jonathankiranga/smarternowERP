Security Model

  • Users are assigned to roles.
  • Roles are assigned permission tokens.
  • Each business function requires a token.
  • Unauthorized access is blocked at runtime.

Administration Areas

Users and Access

  • System Administrator -> Transactions -> Users Maintenance
  • System Administrator -> Transactions -> Access Permissions Maintenance
  • System Administrator -> Transactions -> Page Security Settings

System Configuration

  • System Administrator -> Maintenance -> System Parameters
  • Control defaults for numbering, posting restrictions, display, and communication settings.
  • Manage output preferences such as report identity assets and reporting directories.

Audit and Governance

  • Use System Administrator -> Reports -> View Audit Trail for change review.
  • Use period and role governance to preserve financial control boundaries.
