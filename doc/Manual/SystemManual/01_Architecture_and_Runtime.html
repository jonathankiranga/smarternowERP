Application Shape

The system is a menu-driven ERP workspace with role-filtered options and a central working pane.

  • Users log in and see only permitted module areas.
  • Each module is grouped into Transactions, Inquiries and Reports, and Set Up.
  • The homepage acts as a live operational work queue.

Core Runtime Behavior

Session and Authentication

  • User authentication controls access to all business functions.
  • Session lifetime and execution limits are centrally configured.
  • Every business form submission is validated with a form token.

Authorization

  • Access is permission-token based.
  • Business roles are mapped to permission tokens.
  • Each function is mapped to required permission level.

Data and Transactions

  • All business transactions run on MySQL with transaction commit/rollback support.
  • Operational numbering is centrally managed per document type.
  • Data change operations are audit-trail aware when auditing is enabled.
